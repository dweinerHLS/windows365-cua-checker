PowerShell Prerequisites
SETUP
Complete these steps before running any remediation commands below.
  1. Install PowerShell 7+ — Windows PowerShell 5.1 (built-in) does NOT support the Microsoft Graph module reliably. Download PowerShell 7 from aka.ms/install-powershell or run:
    winget install --id Microsoft.PowerShell --source winget
  2. Set Execution Policy — Allow running scripts in your session
  3. Install Microsoft.Graph Module — The Graph PowerShell SDK (handles NuGet automatically)
  4. Disconnect any existing session — Ensures you log in fresh to the correct tenant
  5. Connect to Microsoft Graph — Authenticates to your target tenant (a browser sign-in window will open)
Step 1 — Install PowerShell 7 (if not installed)
winget install --id Microsoft.PowerShell --source winget
Step 2 — Set Execution Policy (run in PowerShell 7)
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser -Force
Step 3 — Install Microsoft.Graph Module
Install-Module Microsoft.Graph -Scope CurrentUser -Repository PSGallery -Force -AllowClobber
Step 4 — Verify Module Installed
Get-InstalledModule Microsoft.Graph | Format-List Name, Version
Step 5 — Disconnect Existing Session & Connect to Target Tenant
Disconnect-MgGraph -ErrorAction SilentlyContinue Connect-MgGraph -TenantId "YOUR_TENANT_ID" -Scopes "Application.ReadWrite.All","Application-RemoteDesktopConfig.ReadWrite.All","Group.ReadWrite.All","Directory.Read.All","RoleManagement.ReadWrite.Directory","DeviceManagementConfiguration.Read.All"
Step 6 — Verify Connection
Get-MgContext | Format-List Account, TenantId, Scopes
